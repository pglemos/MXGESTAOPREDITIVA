# Matriz de paridade — Módulo Gerencial

## Regra de precedência

1. Especificação funcional textual e Prompt Mestre V2: fórmulas, regras, permissões, estados, critérios de aceite, segurança e auditoria.
2. Base44 autenticado e ZIP de referência: composição visual, textos não conflitantes, layout, estados e comportamento observável.
3. MX atual: React/TypeScript, autenticação, membership, Supabase, RLS, auditoria, serviços canônicos e tokens.

O sidebar escuro do MX é a única exceção visual autorizada. Rotina do Dia usa `CalendarClock` e Mentor Gerencial usa `BrainCircuit`; o restante do shell MX é preservado.

## Baseline auditado em 2026-07-14

- Base44 autenticado: `https://mx-gerente.base44.app`.
- Código de referência: snapshot externo extraído somente para auditoria, conforme `docs/qa/browser-capability-check.md` (não importado nem versionado no bundle).
- MX local: `http://127.0.0.1:3001` (porta descoberta quando `3000` já estava ocupada).
- MX produção: `https://mxperformance.vercel.app`.
- Ferramenta: Chrome DevTools MCP em Chrome real; fallback Playwright/Chromium configurado para E2E.
- Evidências: `docs/qa/evidence/manager-parity/2026-07-14/{base44,mx-local,mx-production}/`.
- As capturas `00-*` e `02-*` sem conteúdo específico são diagnóstico; spinner não é evidência de aprovação. As capturas Base44 `06-*` foram refeitas após espera até ausência de spinner e são as baselines carregadas desta rodada.
- Validação browser local desta retomada: dez rotas renderizadas em `1440×900`, `768×1024` e `390×844`; modais críticos abriram com massa real; o Ranking confirmou os quatro períodos; após adicionar `id/name` aos filtros, o reload do Ranking ficou sem console issue e manteve network `200` escopado por `store_id`.
- E2E autenticado histórico desta retomada: comando `npm run test:e2e -- src/test/manager-module.playwright.ts`, projetos Chromium e mobile-chrome, `5/5` em cada projeto, com credencial carregada somente em runtime. A reexecução final depende de `E2E_ROLE_PASSWORD` ou `E2E_AUTH_PASSWORD` no ambiente; sem a variável, a suíte falha explicitamente. O teste de console/network mantém 4xx/5xx como falha e ignora apenas cancelamentos esperados de navegação mobile (`net::ERR_ABORTED`).
