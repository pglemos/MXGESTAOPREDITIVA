# Wave 4 UX Brief - Desenvolvimento de Pessoas

**Destino:** @ux-design-expert  
**Stories:** DEV-24, DEV-25, DEV-26, DEV-27  
**Status:** Draft

## Objetivo de UX

Transformar a percepcao de "aulas soltas" em uma area de evolucao profissional. O usuario deve entender rapidamente:

- o que precisa fazer agora;
- o que pode estudar livremente;
- o que foi recomendado pelo gerente/PDI/feedback;
- o que ja evoluiu.

## Principios

- Vendedor em mobile primeiro.
- Biblioteca deve ser rapida de buscar.
- Recomendacao precisa explicar o motivo.
- Trilha obrigatoria deve ser separada de conteudo opcional.
- Gerente precisa enxergar acao, nao apenas relatorio.

## Estrutura Sugerida

### Vendedor

Abas ou secoes:

1. Hoje
2. Recomendados
3. Biblioteca
4. Meu progresso
5. PDI/Feedback

### Gerente

Abas ou secoes:

1. Equipe
2. Pendencias
3. Trilhas
4. Biblioteca
5. PDI e feedback

### Admin MX

Abas ou secoes:

1. Conteudos
2. Sugestoes
3. Avaliacoes
4. Trilhas
5. Auditoria

## Estados Obrigatorios

- Sem conteudos recomendados.
- Conteudo recomendado por feedback.
- Conteudo recomendado por PDI.
- Trilha nao iniciada.
- Trilha em andamento.
- Trilha concluida com acao para gerente.
- Conteudo sem avaliacao.
- Sugestao enviada.

## Microcopy

Evitar textos de curso generico. Preferir linguagem de evolucao comercial:

- "Recomendado para seu proximo passo"
- "Lacuna trabalhada"
- "Trilha de entrada"
- "Conteudo livre"
- "Concluir e avisar gerente"

## Riscos de UX

- Colocar tudo em uma unica tela longa pode esconder prioridades.
- Ranking/gamificacao aqui pode competir com a Onda 3; manter simples.
- Trilha obrigatoria nao deve parecer punicao para vendedor experiente.
- Estrelas de avaliacao devem avaliar conteudo, nao vendedor.
