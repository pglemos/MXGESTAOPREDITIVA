# Acessibilidade (WCAG 2.1 AA) — Guia para Contribuidores

> Story 3.11 (UX-015) — Sprint 3 hardening

## TL;DR

- `eslint-plugin-jsx-a11y` está ativo em CI (`.github/workflows/eslint-a11y.yml`).
- Rodar local: `npm run lint:a11y`.
- **PRs com novos errors a11y são bloqueados.** Warnings são toleradas (tracking).

## Regras como `error` (CI bloqueia)

| Regra | Por quê |
|------|---------|
| `jsx-a11y/alt-text` | `<img>` sem `alt` quebra leitores de tela |
| `jsx-a11y/anchor-has-content` | Link vazio = link inacessível |
| `jsx-a11y/anchor-is-valid` | `href="#"` ou inexistente — use `<button>` |
| `jsx-a11y/aria-props` | `aria-*` inválido = atributo morto |
| `jsx-a11y/aria-role` | Role inválido idem |
| `jsx-a11y/aria-unsupported-elements` | Aria em element que não suporta |
| `jsx-a11y/role-has-required-aria-props` | Role completo (ex: `role="checkbox"` precisa `aria-checked`) |
| `jsx-a11y/role-supports-aria-props` | Aria correto para o role |
| `jsx-a11y/heading-has-content` | `<h*>` vazio é landmark quebrado |
| `jsx-a11y/iframe-has-title` | iframe sem title é navegação cega |
| `jsx-a11y/img-redundant-alt` | `alt="image of X"` repete contexto |
| `jsx-a11y/no-redundant-roles` | `<ul role="list">` é ruído |

## Regras como `warn` (tracking, não bloqueia)

| Regra | Motivo de não-bloqueio |
|------|------------------------|
| `jsx-a11y/label-has-associated-control` | Muitos falsos positivos com Radix / inputs custom. **Promover a error em Sprint 4** após cleanup (UX-015B). |
| `jsx-a11y/click-events-have-key-events` | Captura padrões de `<div onClick>` que aparecem muito em design system. Refactor incremental. |
| `jsx-a11y/no-static-element-interactions` | Idem. |
| `jsx-a11y/no-autofocus` | Em telas de login/onboarding é UX consciente — avaliar caso-a-caso. |

## Override pontual

Use `// eslint-disable-next-line jsx-a11y/{rule}` com comentário justificando. Exemplo:

```tsx
{/* eslint-disable-next-line jsx-a11y/no-autofocus -- onboarding: foco intencional no primeiro campo */}
<input autoFocus />
```
