# Design: Refactoring for Supabase Data Integration

## 1. Objective
Refactor the application from mock data (`src/lib/mock-data.ts`) to real-time data from Supabase.

## 2. Architecture
- **State Management**: `src/stores/main.ts` will be updated to hold real data.
- **Data Source**: Direct connection to Supabase using `src/lib/supabase.ts`.
- **Initialization**: Components will trigger a data fetch via `useEffect` upon mounting.

## 3. Scope of Verification
We must validate that all data mapped in `ARQUIVOS_MX/Sistema de Gestão de Alta Performance (1).xlsx` matches the Supabase tables generated by the migration scripts.

## 4. Implementation Plan (Abordagem A)
- **Step 1: Data Verification**
  - Read Excel file structure (using `xlsx` node lib or manual inspection) and cross-reference with Supabase table schemas via `scripts/inspect-supabase-data.ts`.
- **Step 2: Store Refactoring**
  - Update `src/stores/main.ts` to include `isLoading` and `error` states.
  - Implement async actions to fetch data from Supabase.
- **Step 3: Component Integration**
  - Update dashboard views (admin, dono, gerente, vendedor) to use the new store methods and handle loading states.

## 5. Testing Strategy
- Manual browser validation of every tab/screen.
- Verify JSON structure against the expected data from the spreadsheet.
