# Checklist: Lesson Quality

**Version:** 1.0
**Agent:** merrill-designer + rosenshine-teacher
**When to use:** After lesson design task, before content production
**Pass threshold:** 8/10 items minimum

---

## Items

- [ ] **Hook present** — Engaging opening that activates prior knowledge | PASS criteria: Clear hook/attention-getter in first 2 minutes
- [ ] **Problem-centered** — Lesson organized around authentic problem or task | PASS criteria: Problem statement clear, relevant to learners
- [ ] **Activation phase** — Prior knowledge explicitly activated | PASS criteria: Activity or prompt that recalls relevant prior learning
- [ ] **Demonstration** — New skill/concept demonstrated with worked example | PASS criteria: Step-by-step demonstration with narration
- [ ] **Application phase** — Learners practice with guidance | PASS criteria: Guided practice activity with scaffolding
- [ ] **Integration phase** — Learners apply to new context or problem | PASS criteria: Transfer task or authentic application
- [ ] **Review of previous learning** — Connections to prior lessons explicit | PASS criteria: Opening reviews prior lesson key points
- [ ] **Small steps with understanding checks** — Content chunked with frequent checks | PASS criteria: No more than 10-minute segments without check for understanding
- [ ] **Scaffolding and gradual release** — Support decreases from demo to independent | PASS criteria: Clear I do → We do → You do progression
- [ ] **Closure and reflection** — Lesson ends with summary and reflection prompt | PASS criteria: Summary of key points + reflection question

## Veto Conditions (auto-FAIL)
- No demonstration or worked example
- No guided practice
- Lecture exceeds 15 minutes without interaction
- No check for understanding

## Scoring
- **PASS:** 8/10 or more
- **WARN:** 6-7/10 (functional but not optimized)
- **FAIL:** 5/10 or fewer
