# Checklist: Curriculum Architecture Quality

**Version:** 1.0
**Agent:** wiggins-architect
**When to use:** After curriculum architecture task, before lesson design
**Pass threshold:** 7/9 items minimum

---

## Items

- [ ] **Outcomes defined first** — Learning outcomes written before content or activities | PASS criteria: All modules have outcomes in place before learning plan
- [ ] **Essential questions present** — Big questions that frame the learning | PASS criteria: 2-5 essential questions per module that are open-ended and provocative
- [ ] **Assessments aligned** — All assessments map to specific learning outcomes | PASS criteria: Assessment matrix shows 1:1 mapping
- [ ] **Transfer tasks identified** — Authentic tasks requiring application of knowledge | PASS criteria: Each module has at least 1 transfer task (LTEM tier 5+)
- [ ] **WHERETO structure** — Learning plan follows Wiggins WHERETO framework | PASS criteria: All 7 elements present (Where/Hook/Explore/Rethink/Exhibit/Tailor/Organize)
- [ ] **Backward design evident** — Design flow is outcomes → assessment → instruction | PASS criteria: Documentation shows backward design process
- [ ] **Understanding goals** — What students will UNDERSTAND, not just know | PASS criteria: Understanding goals use "students will understand that..." format
- [ ] **Big ideas explicit** — Core concepts that transfer beyond the course | PASS criteria: 3-5 big ideas identified and connected to essential questions
- [ ] **Differentiation planned** — Multiple pathways or options for diverse learners | PASS criteria: At least 2 differentiation strategies per module

## Veto Conditions (auto-FAIL)
- Learning outcomes missing or vague
- No assessments defined
- Activities designed before outcomes
- No essential questions

## Scoring
- **PASS:** 7/9 or more
- **WARN:** 5-6/9 (needs UbD alignment work)
- **FAIL:** 4/9 or fewer
