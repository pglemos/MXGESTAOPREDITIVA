# Framework Primario: 4R Model + Skills-Based Organization

**Type:** Primary Operating Framework
**Agent:** @kaizen:josh-bersin
**Status:** Validado com pesquisa em 800+ organizacoes globais

## Overview

O sistema operacional de Josh Bersin e construido sobre duas camadas: o modelo 4R como ferramenta decisoria de workforce planning, e a Skills-Based Organization como arquitetura-alvo. O 4R responde "o que fazer agora com um gap." O SBO responde "como organizar o trabalho para o futuro." Juntos, transformam HR de funcao administrativa em inteligencia estrategica.

## Framework 1: The 4R Model

### As 4 Opcoes

```text
Capability Gap Identificado
    |
    v
[1. REDESIGN] --> O trabalho precisa existir neste formato?
    |                - Automacao/AI pode absorver?
    |                - Pode redistribuir entre roles existentes?
    |                - Job design precisa modernizacao?
    |
    v
[2. RESKILL] --> Alguem interno pode aprender?
    |               - Skills adjacentes existem (70%+ overlap)?
    |               - Ha tempo (3-12 meses)?
    |               - Infraestrutura de learning existe?
    |
    v
[3. RETAIN] --> Estamos perdendo quem ja sabe?
    |              - Turnover acima do benchmark?
    |              - Conhecimento tacito em risco?
    |              - Custo de reposicao: 1.5-2x salario
    |
    v
[4. RECRUIT] --> Precisamos de alguem de fora?
                   - Skill completamente nova?
                   - Urgencia alta (< 3 meses)?
                   - Score interno < 0.3?
```

### Regra de Ouro

A ordem de avaliacao e CONTRA-INTUITIVA: comece por Redesign (mais estrategico), termine em Recruit (mais caro e arriscado). A maioria das organizacoes faz o inverso — abre vaga primeiro, pensa depois.

### Criterios de Decisao por Opcao

#### REDESIGN — Quando Escolher

| Criterio | Threshold |
|----------|-----------|
| Automacao viavel | > 50% da funcao automatizavel |
| Necessidade temporaria | Projeto com fim definido |
| Job design obsoleto | JD sem atualizacao > 12 meses |
| Redistribuicao possivel | 2+ roles podem absorver parcialmente |

**Metricas de sucesso:** automation-rate, productivity-per-FTE, work-fragmentation-score

#### RESKILL — Quando Escolher

| Criterio | Threshold |
|----------|-----------|
| Adjacencia de skills | > 70% overlap com skill-alvo |
| Janela de tempo | 3-12 meses disponiveis |
| Infra de learning | Plataforma ou programa existente |
| ROI vs. recruit | Custo reskill < custo recruit + onboarding |

**Metricas de sucesso:** skill-acquisition-rate, time-to-proficiency, reskilling-ROI, internal-mobility-rate

#### RETAIN — Quando Escolher

| Criterio | Threshold |
|----------|-----------|
| Turnover em roles criticos | > 15% ao ano |
| Risk indicators | Desengajamento, ofertas externas |
| Conhecimento tacito | Alto e nao documentado |
| Custo de substituicao | > 1.5x salario anual |

**Metricas de sucesso:** voluntary-turnover, engagement-score, eNPS, stay-interview-insights

#### RECRUIT — Quando Escolher

| Criterio | Threshold |
|----------|-----------|
| Skill inexistente internamente | Score interno < 0.3 |
| Urgencia critica | < 3 meses para impacto |
| Mercado tem oferta | Candidatos qualificados disponiveis |
| Nenhuma opcao interna viavel | Todas as 3 opcoes anteriores avaliadas |

**Metricas de sucesso:** time-to-fill, quality-of-hire, cost-per-hire, new-hire-retention-90d

## Framework 2: Skills-Based Organization (SBO)

### O Modelo de Maturidade (5 Niveis)

| Nivel | Nome | Indicador-Chave | Foco |
|-------|------|-----------------|------|
| 0 | Traditional | JDs fixas, hierarquia rigida | Sobrevivencia |
| 1 | Awareness | Lideranca reconhece limitacoes | Business case |
| 2 | Foundation | Skills taxonomy implementada | Inventario |
| 3 | Building | Talent marketplace ativo | Mobilidade |
| 4 | Operating | Decisoes baseadas em skills data | Inteligencia |
| 5 | Optimizing | Predicao proativa de gaps | Inovacao |

### Implementacao Step-by-Step

**Step 1: Skills Inventory (Mes 1-2)**
- Mapear skills de todos os membros
- Usar framework: Technical Skills + Behavioral Skills + Domain Skills
- Validar com gestores e auto-avaliacao cruzada
- Output: Inventario com proficiency levels (1-5)

**Step 2: Skills Taxonomy (Mes 2-3)**
- Criar linguagem padronizada de skills
- Alinhar com taxonomias de industria quando possivel
- Maximo 100-200 skills para organizacoes < 1000 pessoas
- Output: Taxonomia versionada e governada

**Step 3: Gap Analysis (Mes 3-4)**
- Cruzar skills existentes vs. skills necessarias (estrategia)
- Identificar gaps por funcao, departamento e organizacao
- Priorizar gaps por impacto no negocio
- Output: Mapa de gaps com priorizacao

**Step 4: Talent Marketplace (Mes 4-6)**
- Implementar mecanismo de matching interno
- Projetos e oportunidades visiveis para toda organizacao
- Career pathways baseados em adjacencias de skills
- Output: Plataforma ou processo de mobilidade interna

**Step 5: Continuous Learning Integration (Mes 6-9)**
- Learning integrado ao fluxo de trabalho (nao evento separado)
- Programas de reskilling conectados a gaps identificados
- Mentoring e peer learning como complemento
- Output: Learning ecosystem operando

**Step 6: Workforce Intelligence (Mes 9-12)**
- Dashboards de skills em tempo real
- Predicao de gaps futuros baseada em estrategia + mercado
- Metricas de impacto: mobilidade, retencao, time-to-proficiency
- Output: Decision-making data-driven para workforce

## Framework 3: The Irresistible Organization

### Os 5 Pilares

```text
PILAR 1: MEANINGFUL WORK
  - Autonomia nas decisoes do dia a dia
  - Conexao clara entre trabalho individual e impacto organizacional
  - Trabalho desafiador que usa e desenvolve skills
  - Metrica: "Meu trabalho faz diferenca" (survey)

PILAR 2: STRONG MANAGEMENT
  - Gestores como coaches, nao como supervisores
  - Feedback continuo, nao review anual
  - Gestores capacitados em career development conversations
  - Metrica: Manager effectiveness score

PILAR 3: POSITIVE WORK ENVIRONMENT
  - Cultura de confianca e seguranca psicologica
  - Flexibilidade (quando, onde e como trabalhar)
  - Inclusao genuina, nao apenas diversidade numerica
  - Metrica: Belonging index

PILAR 4: GROWTH OPPORTUNITY
  - Mobilidade interna facilitada e incentivada
  - Learning continuo acessivel a todos
  - Career pathways multiplos (tecnico, gestao, especialista)
  - Metrica: Internal mobility rate, learning hours

PILAR 5: TRUST IN LEADERSHIP
  - Transparencia em decisoes e direcao
  - Comunicacao autentica e frequente
  - Coerencia entre discurso e acao
  - Metrica: Leadership trust score, eNPS
```

## Integration: Como os Frameworks se Conectam

```text
Skills-Based Organization (arquitetura) ->
  4R Model (decisoes de workforce) ->
    Irresistible Organization (experiencia) ->
      Workforce Intelligence (feedback loop)
```

SBO define COMO organizar. 4R define O QUE FAZER com gaps. Irresistible define POR QUE pessoas ficam e prosperam. Workforce Intelligence fecha o loop com dados para decisoes melhores. Nenhum funciona isolado.
