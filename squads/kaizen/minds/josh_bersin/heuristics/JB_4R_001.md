# JB_4R_001 — 4R Decision Matrix

**Type:** Decision Heuristic
**Phase:** 1 (Workforce Planning)
**Agent:** @kaizen:josh-bersin
**Pattern:** JB-CORE-001 (4R Model)

## Purpose

Framework decisorio para quando uma organizacao (ou squad) identifica um capability gap. Em vez de default para "contratar", o modelo forca a avaliacao sistematica de 4 opcoes: Recruit, Retain, Reskill, Redesign. A ordem de avaliacao e invertida ao senso comum — comeca por Redesign e termina em Recruit.

## Configuration

```yaml
JB_4R_001:
  name: "4R Decision Matrix"
  phase: 1
  pattern_reference: "JB-CORE-001"

  weights:
    redesign_priority: 0.9        # avaliar primeiro — o trabalho precisa existir assim?
    reskill_potential: 0.85       # segundo — alguem interno pode aprender?
    retain_risk: 0.8              # terceiro — estamos perdendo quem ja sabe?
    recruit_necessity: 0.7        # ultimo recurso — so se interno nao resolve

  thresholds:
    skill_adjacency_minimum: 0.7        # 70%+ overlap para considerar reskill viavel
    time_to_proficiency_max_months: 12  # maximo aceitavel para reskilling
    turnover_risk_threshold: 0.15       # acima de 15% turnover, ativar retencao
    automation_feasibility_minimum: 0.5 # 50%+ automatizavel = considerar redesign
    recruit_only_if_internal_score: 0.3 # so recrutar se score interno < 0.3

  veto_conditions:
    - condition: "recruit_without_evaluating_internal"
      action: "VETO — Avalie Redesign, Reskill e Retain ANTES de abrir vaga externa."
    - condition: "reskill_without_adjacency_mapping"
      action: "VETO — Mapeie skills adjacentes antes de iniciar programa de reskilling."
    - condition: "retain_with_only_money"
      action: "VETO — Retencao so com aumento salarial nao funciona. Enderece meaningful work, growth e management."
    - condition: "redesign_without_impact_analysis"
      action: "VETO — Antes de redesenhar roles, analise impacto em produtividade, moral e interdependencias."

  decision_tree:
    - IF capability_gap_identified THEN evaluate_redesign_first
    - IF work_can_be_automated_or_redistributed THEN REDESIGN
    - IF automation_not_viable AND internal_adjacency >= 0.7 THEN RESKILL
    - IF skills_exist_internally AND turnover_risk > 0.15 THEN RETAIN
    - IF no_internal_option_viable THEN RECRUIT
    - IF multiple_options_viable THEN combine_strategies (ex: reskill + recruit)
    - TERMINATION: capability_gap_closed_with_measured_outcome

  output:
    type: "workforce decision + action plan"
    values: ["REDESIGN", "RESKILL", "RETAIN", "RECRUIT", "HYBRID"]
```

## O Modelo 4R em Detalhe

```text
OPCAO 1: REDESIGN (Avaliar Primeiro)
  Pergunta-chave: "O trabalho precisa existir neste formato?"
  Cenarios:
    - Automacao/AI pode absorver a funcao
    - Trabalho pode ser redistribuido entre roles existentes
    - Skill e temporaria (projeto especifico)
    - Job design antiquado que precisa modernizacao
  Exemplo: Em vez de contratar 3 data analysts, implementar
    dashboards self-service + reskill gestores em data literacy

OPCAO 2: RESKILL (Avaliar Segundo)
  Pergunta-chave: "Alguem aqui dentro pode aprender isso?"
  Cenarios:
    - Colaboradores tem 70%+ de skills adjacentes
    - Janela de tempo permite desenvolvimento (3-12 meses)
    - Organizacao tem infraestrutura de learning
    - Custo de reskill < custo de recruit + onboarding
  Exemplo: Developers Python sendo reskilled para ML Engineering
    (adjacencia alta, custo menor que contratar ML Engineers senior)

OPCAO 3: RETAIN (Avaliar Terceiro)
  Pergunta-chave: "Estamos perdendo quem ja tem essas skills?"
  Cenarios:
    - Turnover em roles criticos acima do benchmark
    - Colaboradores-chave mostrando sinais de desengajamento
    - Conhecimento tacito seria perdido com saida
    - Custo de reposicao e 1.5-2x salario anual
  Exemplo: Tech Lead senior com 5 anos de contexto institucional
    pensando em sair — retain com projeto desafiador + mentoring role

OPCAO 4: RECRUIT (Ultimo Recurso)
  Pergunta-chave: "Precisamos de alguem de fora?"
  Cenarios:
    - Skill completamente nova para a organizacao
    - Urgencia alta (< 3 meses para impacto)
    - Nenhuma opcao interna viavel (score < 0.3)
    - Mercado tem oferta de profissionais qualificados
  Exemplo: Primeira contratacao de especialista em AI Safety —
    ninguem interno tem a base, mercado tem candidatos
```

## Application

**Input:** Descricao de um capability gap identificado.

**Process:**
1. Defina o gap com precisao: qual skill falta? Para qual objetivo?
2. Avalie REDESIGN: o trabalho pode ser reestruturado ou automatizado?
3. Avalie RESKILL: mapeie adjacencias internas (quem tem 70%+ das skills?)
4. Avalie RETAIN: ha risco de perder quem ja sabe?
5. Avalie RECRUIT: so se score interno < 0.3
6. Recomende a combinacao otima (frequentemente e hibrida)

## Examples

### Gap: Squad precisa de capacidade de analise de dados

- **REDESIGN:** Implementar dashboards self-service, reduzir necessidade de analista dedicado
- **RESKILL:** Dev backend com SQL forte -> data analytics em 4 meses
- **RETAIN:** Analista jr existente mostrando burnout -> redistribuir carga + mentoria
- **RECRUIT:** So se nenhuma opcao acima resolver em tempo habil
- **Decisao:** REDESIGN (dashboards) + RESKILL (dev backend) = gap fechado sem contratacao

### Gap: Organizacao precisa de expertise em AI/ML

- **REDESIGN:** Avaliar quais casos de uso podem usar AI as-a-service (sem ML engineers)
- **RESKILL:** Developers senior com background em estatistica -> ML Engineering (6-9 meses)
- **RETAIN:** Data Scientists existentes recebendo ofertas externas -> projeto de inovacao + equity
- **RECRUIT:** ML Architect senior para liderar a transformacao (nao existe internamente)
- **Decisao:** HYBRID — Recruit 1 lider + Reskill 3 devs + Redesign casos simples para SaaS

## Diagnostic Questions

1. "Qual e exatamente o capability gap? (skill, nao job title)"
2. "O trabalho pode ser redesenhado ou automatizado?"
3. "Quem internamente tem skills adjacentes (70%+ overlap)?"
4. "Estamos em risco de perder quem ja tem essas skills?"
5. "Qual e a urgencia real? (semanas, meses, trimestre)"
